### Summary: What's in the computation graph

**ACTIVE (receives gradients):**
- L4 ring (feedforward processing)
- PV pool (divisive normalization)
- L2/3 ring (W_rec, w_som, w_pv_l23 all active)
- SOM ring (integrates negative part of V2 feedback)
- V2 GRU + head_mu + head_pi + head_feedback
- DeepTemplate (only through energy cost — weak gradient signal)
- feedback_scale buffer (not a parameter, set externally during warmup)

**DEAD (no gradients, wasted parameters):**
- VIP ring — hardcoded to zero, never called
- EmergentFeedbackOperator — entirely bypassed (all alpha_inh, alpha_vip, alpha_apical, som_baseline, som_tonic, vip_baseline)
- w_template_drive — never used
- w_vip_som — never used
- Apical gain pathway — never used (apical_gain=None)

**COMPUTED BUT WEAKLY CONNECTED:**
- DeepTemplate — computed every step, stored in state, enters energy cost.
  But NOT used for L2/3 modulation. Gradient flows only through
  `outputs["deep_template"].abs().mean()` in energy_cost(). This is a weak
  L1 signal that pushes deep_template toward zero.

### Feedback signal flow (simple_feedback)

```
V2 GRU hidden state
    │
    ├──→ head_mu → softmax → q_pred ──→ deep_template (→ energy cost only)
    │                              └──→ loss functions (prior_kl, sensory)
    ├──→ head_pi → softplus → pi_pred → pi_pred_eff (→ deep_template only)
    │
    └──→ head_feedback → feedback_signal [B,N]
              │
              ├──× feedback_scale → scaled_fb
              │
              ├──→ relu(scaled_fb) → center_exc → L2/3 excitatory drive
              │
              └──→ relu(-scaled_fb) → som_drive → SOM ring → L2/3 inhibition
```

The feedback pathway is a single learned linear head (head_feedback: 16→36)
that outputs a raw signal, split by sign into excitation (positive) and
SOM inhibition (negative). This is much simpler than the full
EmergentFeedbackOperator (3 circulant kernels + delta-SOM + tonic + VIP).
